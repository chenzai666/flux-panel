# 从稳定版迁移到 Beta 版指南

## 版本说明

- **稳定版**（1.x）：使用 MySQL 数据库
- **Beta 版**（2.x）：使用 SQLite 数据库，支持转发链（多跳中转）和负载均衡

由于数据库类型不同，不能直接原地升级，需要手动迁移数据。

---

## 方法一：全新部署（推荐）

如果转发规则不多，建议直接使用 Beta 版全新部署，在管理面板中重新创建隧道和转发。

### 部署步骤

1. 下载 `docker-compose-v4.yml`（IPv4）或 `docker-compose-v6.yml`（IPv6）
2. 创建 `.env` 文件：
   ```env
   JWT_SECRET=你的密钥（至少32位随机字符串）
   BACKEND_PORT=6365
   FRONTEND_PORT=80
   ```
3. 启动服务：
   ```bash
   docker compose -f docker-compose-v4.yml up -d
   ```

---

## 方法二：数据导出迁移

### 从稳定版导出

在稳定版的**转发管理**页面，使用**导出**功能，选择要迁移的隧道，复制导出的文本。

导出格式为：`目标地址|转发名称|入口端口`，每行一个。

### 在 Beta 版中导入

1. 先在 Beta 版中创建对应的隧道（需要重新配置节点和转发链）
2. 在**转发管理**页面使用**导入**功能，选择对应隧道，粘贴导出的数据

---

## Beta 版新功能说明

### 转发链（隧道转发）

Beta 版支持多跳中转链路，即：

```
用户 → 入口节点 → [中转节点1] → [中转节点2] → ... → 出口节点 → 目标
```

创建隧道时，类型选择**隧道转发**，即可配置：
- **入口节点**：接收用户连接的节点（支持多个，负载均衡）
- **转发链**：中间中转节点（可选，支持多跳，每跳支持多个节点负载均衡）
- **出口节点**：连接目标地址的节点（支持多个，负载均衡）

### 负载均衡策略

每个节点组支持以下负载均衡策略：
- `fifo`：主备模式，优先用第一个，故障时切换
- `round`：轮询模式，依次使用每个节点
- `rand`：随机模式，随机选择节点
- `hash`：IP哈希，同一客户端IP始终路由到同一节点

---

## 注意事项

- Beta 版的用户、节点配置无法从稳定版直接迁移，需要重新创建
- Beta 版隧道结构发生根本性变化（chain_tunnel 表替代原来的 in_node_id/out_node_id 字段），因此数据库层面无法直接迁移隧道配置
- 转发数据可以通过导出/导入功能迁移（需要先在 Beta 版创建好对应隧道）
